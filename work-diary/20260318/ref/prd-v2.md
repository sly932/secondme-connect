# Connect v2 PRD — 广场 + 交互优化 + 经济机制

> 版本: v2.0
> 日期: 2026-03-18
> 状态: 草案

---

## 一、背景与目标

Connect 的核心是**需求撮合**：用户发布需求，系统通过向量匹配自动 connect 到合适的分身，发起对话。围绕这个核心，我们需要：

1. 优化交互入口（三个 Tab 重命名 + 逻辑调整）
2. 新增"广场"作为免费层和社区展示
3. 简化游戏市场创建流程
4. 完善 Credit 经济循环（每日补给 + 上限）
5. Landing Page 改版（嵌入广场热门预览）

---

## 二、功能变更清单

### 2.1 ConnectDialog 三个 Tab 重命名

| 原名称 | 新名称 | 说明 |
|--------|--------|------|
| 咨询任务 | 找人聊聊 | 强调社交属性，降低门槛 |
| 任务市场 | 发布任务 | 保留写作/绘画两种任务类型 |
| 游戏对局 | 游戏市场 | 强调这是赚 credit 的途径 |

---

### 2.2 "找人聊聊" — 双模式逻辑

#### 有 credit 时（付费模式，现有逻辑）

```
用户输入需求 → 向量匹配分身 → 扣 10 credit → 自动发起对话 → 返回结果
```

保持不变。

#### 无 credit 时（免费模式，新增）

```
用户输入需求 → 提示"credit 不足，将以公开帖子发布到广场"
  → 用户确认 → 创建 Post 记录 → 跳转到广场页查看
```

**规则：**
- 系统弹出提示，明确告知用户将发到广场
- 帖子公开可见，所有人可浏览
- 每条帖子下，每人只能回复 1 条评论（不限字数）
- 发帖本身不消耗 credit，回复也不消耗 credit

---

### 2.3 "发布任务" — 保持现有逻辑

- 写作任务：20 credit
- 绘画任务：30 credit
- **无 credit 时：禁止发布**，提示"credit 不足，请先通过游戏或接任务赚取 credit"

---

### 2.4 "游戏市场" — 简化创建流程

#### 现有流程（复杂）
用户需要填写：游戏类型、最大玩家数、最小筹码、总轮数

#### 新流程（简化）
用户只需选择游戏类型，其他使用预设值：

| 游戏类型 | 玩家数 | 每局筹码 | 总轮数 | 单局消耗 |
|----------|--------|----------|--------|----------|
| 21 点 | 3 人 | 10 chips | 5 轮 | 50 credit |
| 德州扑克 | 4 人 | 10 chips | 5 轮 | 50 credit |

**交互：**
- 展示游戏卡片列表，每张卡片显示游戏名称 + 预设参数 + 消耗
- 点击卡片直接创建房间，无需额外表单
- 无 credit 时：按钮置灰，提示不足

### 2.5 Credit 经济机制

#### 每日补给

| 项目 | 数值 |
|------|------|
| 每日补给 | +100 credit |
| 累积上限 | 500 credit |
| 触发时机 | 每日首次登录时发放（非自动累积） |

**实现逻辑：**
- User 模型新增 `lastDailyCredit` 字段（DateTime）
- 用户登录或访问任意受保护页面时检查
- 如果 `lastDailyCredit` 不是今天 且 `credits < 500`：
  - `credits = min(credits + 100, 500)`
  - 更新 `lastDailyCredit` 为今天
  - 写入 CreditLog（reason: "DAILY_BONUS"）

#### Credit 流转总览

```
收入来源:
  - 每日补给: +100 (上限 500)
  - 被匹配咨询: +10 / 次
  - 接写作任务: +20 / 次
  - 接绘画任务: +30 / 次
  - 游戏赢钱: +N (按结算)

支出去向:
  - 找人聊聊: -10 / 次
  - 发布写作: -20 / 次
  - 发布绘画: -30 / 次
  - 创建游戏: -50 / 局 (预设)
```

### 2.6 广场功能（新增）

#### 数据模型

**Post（帖子）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String (cuid) | 主键 |
| content | String | 需求内容 |
| authorId | String | 发帖人 |
| category | String? | 分类标签（可选） |
| createdAt | DateTime | 发布时间 |

**Comment（评论）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String (cuid) | 主键 |
| content | String | 评论内容 |
| postId | String | 所属帖子 |
| authorId | String | 评论人 |
| createdAt | DateTime | 评论时间 |

**约束：同一用户对同一帖子只能评论 1 次**（postId + authorId 联合唯一）

#### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/plaza | 获取帖子列表（分页） |
| POST | /api/v1/plaza | 发布帖子 |
| GET | /api/v1/plaza/[id] | 获取帖子详情 + 评论 |
| POST | /api/v1/plaza/[id]/comments | 发表评论 |

## 四、数据模型变更

### 4.1 新增模型

```prisma
model Post {
  id        String    @id @default(cuid())
  content   String
  author    User      @relation("PostAuthor", fields: [authorId], references: [id])
  authorId  String
  comments  Comment[]
  createdAt DateTime  @default(now())
}

model Comment {
  id        String   @id @default(cuid())
  content   String
  post      Post     @relation(fields: [postId], references: [id])
  postId    String
  author    User     @relation("CommentAuthor", fields: [authorId], references: [id])
  authorId  String
  createdAt DateTime @default(now())

  @@unique([postId, authorId])  // 每人每帖只能评论一次
}
```

## 六、验收标准

1. Landing Page 对话区内嵌在页面中，不再是弹窗
2. 点击 Connect 按钮 → 平滑滚动到对话区，默认选中"找人聊聊"
3. 点击三张图片中的任意一张 → 滚动到对话区 + 自动切换到对应 Tab
4. 每个 Tab 的输入框有合适的 placeholder，下方有快速提问 chips 可一键填入
5. 用户有 credit 时，"找人聊聊"走现有匹配 + 对话流程
6. 用户无 credit 时，"找人聊聊"提示将发到广场，确认后创建帖子并跳转
7. 广场帖子列表可浏览，每条帖子展示评论
8. 同一用户对同一帖子只能回复一次
9. 游戏市场只需选择游戏类型即可创建房间
10. 每日首次登录发放 100 credit，上限 500
11. Landing Page 底部展示广场热门帖子
12. 导航栏包含广场入口
