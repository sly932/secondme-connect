# Connect · 产品需求文档 v0.2

---

## 一、产品概述

**产品名称**：Connect

**核心定位**：一个以 AI 分身为核心的需求撮合与任务执行平台。用户注册后拥有一个基于自身记忆和性格构建的 AI 分身，分身可以替用户接单赚取 credit，也可以被用户派出去完成咨询或创作任务。

**核心 slogan**：你不在，它在。

---

## 二、用户注册

用户通过 SecondMe OAuth2 完成注册和登录，无需单独设置账号密码。

具体流程是：用户点击登录 → 跳转 SecondMe 授权页 → 用户授权 → 系统获取授权码 → 后端换取 Access Token → 本地绑定用户账号。

需申请的权限范围包括：`user.info`、`user.info.shades`、`user.info.softmemory`、`chat`。

新用户注册完成后，系统自动赠送 **100 credit** 作为初始资金，同时触发分身档案的自动构建流程。

---

## 三、分身档案

用户完成授权后，系统自动调用 SecondMe API 拉取以下数据，在本地数据库构建分身档案：

- 基础信息：姓名、头像、个人简介（bio）
- 兴趣标签（shades）
- 软记忆（softmemory）：用户积累的个人知识库内容

档案构建完成后，展示在用户个人主页，用户可查看自己的分身档案摘要，但不可直接编辑（数据源来自 SecondMe，保持同步）。

档案中同时维护以下运营字段：

- credit 余额
- 接单总数
- 接单总收入
- 历史任务记录

---

## 四、向量化

分身档案构建完成后，系统自动将档案文本内容（bio + 兴趣标签 + 软记忆拼接）做 Embedding 处理，生成向量并存入向量数据库。

向量化在以下情况触发更新：用户首次注册、用户软记忆发生变化（可设置定期同步，例如每 24 小时重新拉取一次）。

向量数据库使用 Supabase（内置 pgvector 扩展），检索方式为余弦相似度，默认返回 Top 10 结果，排除用户自身分身。

Embedding 模型通过 OpenRouter API 调用 OpenAI Embedding 模型。

## 五、需求与任务

产品提供两个独立入口，底层共用同一套向量匹配引擎和 credit 扣费逻辑。

### 5.1 入口一：咨询任务

**用户路径**：用户在首页输入框描述自己的需求（一句话或几句话）→ 系统向量匹配，返回 Top 10 分身 → 根据用户设置（自动或手动）下单 → 系统以匹配分身的身份，通过 SecondMe `chat/stream` 接口自动发起多轮对话 → 对话结果展示给用户。

**交付物**：多个分身的对话回复内容，用户获得多角度的意见、建议或反馈。

**credit 消耗**：按匹配分身数量计费，每个分身的对话消耗固定 credit（建议定价：每个分身 10 credit）。

**下单模式**（用户可在个人设置中配置，也可在发布时临时切换）：

- **自动模式**：匹配完成后系统直接取 Top N 分身自动下单执行，用户无需确认。
- **手动模式**：匹配完成后展示候选分身列表，用户手动选择后点击确认下单。

### 5.2 入口二：任务市场

**用户路径**：用户进入任务发布页，选择任务类型，填写任务描述 → 系统向量匹配返回候选分身 → 根据用户设置（自动或手动）下单 → 分身自动接单执行 → 交付具体成果。

**任务类型**目前支持两类：

**写作任务**：分身调用 `chat/stream` 接口，以自身风格生成文章、文案、策划等文字内容，流式返回给发布方。

**绘画任务**：分身先生成绘画 prompt（调用 `chat/stream`），系统再用该 prompt 调用免费绘图 API（推荐 Pollinations.ai），最终将图片返回给发布方。

**credit 消耗**：写作任务每单消耗 20 credit，绘画任务每单消耗 30 credit。

**接单机制**：接单方永远是分身全自动托管，无需真人介入。前端展示"模拟接单"动效（分身评估中 → 已接单 → 执行中 → 已完成），credit 在"已接单"状态时扣除，任务完成后 credit 自动打入接单方账户。

**任务时长管理**：写作任务超时阈值 2 分钟，绘画任务 3 分钟，超时后任务标记为失败，credit 退还发布方。

## 六、下单模式设置

用户可在个人设置页配置默认下单模式，也可在每次发布时临时切换。

- **自动模式**：系统取匹配 Top N（默认 Top 1，用户可设置 Top 1~5）自动执行，无需确认。
- **手动模式**：展示候选分身列表，用户手动勾选后确认下单。

两种模式下，接单方分身的执行过程均为全自动，与发单方的选择无关。

## 八、双 Token 体系

### 8.1 SecondMe Token

- 通过 OAuth2 获取，用于后端调用 SecondMe API
- Access Token 有效期 2 小时，Refresh Token 30 天
- 用户无感知，由后端自动管理和刷新

### 8.2 App Token（API Key）

- 格式：`ck-` 前缀 + 随机字符串
- 用户首次登录时自动生成
- 长期有效，直到用户主动重新生成
- 用户在设置页可查看、复制、重新生成
- 调用 Connect 开放 API 时必须在 Header 中携带：`Authorization: Bearer ck-xxx`

---

## 九、开放 API

所有核心功能均暴露为可直接调用的 REST API，文档以 Swagger 风格展示在 `/docs` 路径。

主要端点：
- `POST /api/v1/consult` — 发起咨询任务
- `POST /api/v1/tasks` — 发布任务（写作/绘画）
- `GET /api/v1/tasks/:id` — 查询任务状态与结果
- `GET /api/v1/profile` — 获取当前用户分身档案
- `GET /api/v1/credits` — 查询 credit 余额

所有端点需携带 App Token，响应格式统一为 JSON。
